# rust-snake

一个基于 Rust、`ratatui` 和 `crossterm` 的终端贪吃蛇项目，支持多 AI 敌蛇对战。

## 游戏截图

![](docs/screenshot.png)

## 当前功能

- 终端 TUI 界面
- 固定 tick 游戏循环（160ms）
- `WASD` 和方向键控制玩家移动
- `Enter` / `Space` / 方向键开始游戏
- `Space` 暂停/继续
- `r` 重新开始
- `q` 退出
- 玩家蛇吃到食物后增长
- 4 个 AI 敌蛇自动追逐食物
- 敌蛇撞死后会重生并保留分数
- 玩家撞墙、撞自己、或撞到敌蛇后游戏结束
- 超级食物（$）：吃到后额外增长 4 节
- 炸弹（X）：玩家触碰到炸弹游戏结束
- 分数显示（玩家分数 / 敌蛇总分）
- 敌蛇分数和方向显示颜色与蛇自身颜色对应
- 按当前窗口大小初始化棋盘
- 调整窗口后按新尺寸自动开始新游戏
- 终端窗口过小时显示提示
- 支持 `-nocolor` 启动无色版本

## 运行方式

确保本地已经安装 Rust 工具链，然后在项目根目录执行：

```bash
cargo run
```

启动无色版本：

```bash
cargo run -- -nocolor
```

cargo test

## 操作说明

- `W` / `A` / `S` / `D` 或方向键：移动
- `Enter` / `Space` / 方向键：开始游戏
- `Space`：暂停或继续
- `r`：重新开始
- `q`：退出游戏

## 游戏说明

### 角色符号

| 角色 | 符号 | 颜色 |
|------|------|------|
| 玩家蛇头 | `@` | 白色 |
| 玩家蛇身 | `o` | 白色 |
| 敌蛇头 | `A` `B` `C` `D` `E` `F` | 亮洋红/亮青/亮黄/亮红（循环） |
| 敌蛇身 | `a` `b` `c` `d` `e` `f` | 洋红/青/黄/红（循环） |
| 食物 | `*` | 绿色 |
| 超级食物 | `$` | 亮黄色 |
| 炸弹 | `X` | 红色 |
| 空格子 | `·` | 深灰色 |

### 游戏规则

- 玩家蛇(@)与 4 条 AI 敌蛇(A/B/C/D)同屏对战
- AI 敌蛇以 85% 概率追逐最近食物，15% 概率随机漫步（5-15 步）
- 双方都能吃食物(*)增长，食物同时存在 4 颗
- 超级食物($)吃到后额外增长 4 节，同时最多存在 1 颗
- 炸弹(X)触碰后游戏立即结束，同时最多存在 2 颗
- 食物数量会根据棋盘空间动态调整
- 玩家撞墙、撞自己、或撞到敌蛇则游戏结束
- 敌蛇撞死后会立即重生，分数保留，身体化为食物
- 调整窗口大小会自动按新尺寸重新开始

## 游戏逻辑

### 核心 Tick 处理流程

每 160ms 执行一次 `tick()`，游戏逻辑分为五个阶段：

**1. 方向同步**
- 将玩家输入的 `pending_direction` 同步为实际生效的 `direction`
- 直接反向输入（如向上时按向下）会被忽略，防止蛇原地掉头

**2. 玩家移动计算**
- 根据当前位置和方向计算下一步位置
- 判断该位置是否有食物、超级食物或炸弹

**3. AI 移动规划**
- 为每条 AI 敌蛇计算下一步的移动意图
- 所有 AI 的规划在碰撞判断之前完成，确保公平性

**4. 碰撞检测**
- 先判断玩家是否会撞墙、撞自身或撞 AI
- 若玩家死亡，立即结束游戏
- 再判断每个 AI 是否会撞死（撞墙、撞玩家、撞其他AI）
- 头撞头碰撞：体型较小的一方死亡，同体型同死

**5. 状态更新**
- 玩家蛇前进一步，吃食物则增长，否则移除尾巴
- 各 AI 蛇按各自的 plan 前进一步
- 撞死的 AI 在同一帧重生到新位置，分数保留
- 补充被吃掉的食物
- tick 计数器递增

### AI 决策分层策略

AI 敌蛇采用分层决策机制：

| 层级 | 行为 | 触发条件 | 说明 |
|------|------|----------|------|
| 1 | 随机漫步 | `random_walk_steps > 0` | 保持当前漫步方向，每步递减计数器 |
| 2 | 开始新漫步 | 15% 概率 | 随机选择安全方向，漫步 5-15 步 |
| 3 | 追逐食物 | 默认 | 寻找最近食物，优先水平后垂直移动 |
| 4 | 保持方向 | 优先方向不安全 | 尝试保持原方向 |
| 5 | 紧急逃生 | 所有方向危险 | 从所有安全方向中随机选择 |
| 兜底 | 原方向不变 | 无安全方向 | 下一秒可能撞死 |

### 尾巴移动规则

这是贪吃蛇游戏中的关键细节：

- **正常移动**：蛇移动时尾巴会向前移动一格
- **吃食物时**：尾巴**不移动**（食物位置变成新尾巴），蛇身长度 +1
- **判断占用**：当 `grows == true`（要吃食物）时，尾巴位置仍被视为被身体占用

```rust
// 判断逻辑：尾巴位置是否被占用
let is_tail = index == 0;
*segment == position && (grows || !is_tail)
```

### 头撞头碰撞规则

当两条蛇的头部移动到同一位置时：

| 玩家体型 | AI 体型 | 结果 |
|----------|---------|------|
| 较大 | 较小 | AI 死亡 |
| 较小 | 较大 | 玩家死亡 |
| 相等 | 相等 | 双方死亡 |

### 物品机制

| 物品 | 符号 | 增长 | 得分 | 效果 |
|------|------|------|------|------|
| 普通食物 | `*` | +1 节 | +1 分 | 绿色 |
| 超级食物 | `$` | +4 节 | +4 分 | 亮黄色 |
| 炸弹 | `X` | - | - | 触碰立即死亡，红色 |

- 食物同时存在 4 颗，超级食物最多 1 颗，炸弹最多 2 颗
- 食物数量会根据棋盘空间动态调整

### 敌蛇重生机制

敌蛇撞死后：
1. 身体转化为普通食物留在棋盘上
2. 在棋盘角落重生（新位置与原位置无关）
3. 分数**保留**（不归零）

重生位置优先四个角落（左上、右上、左下、右下），每个角落会优先尝试能朝棋盘内部移动的预设形态。
